TCP工作特点

  • 是在不可靠IP层实现的可靠数据传输协议

  • 提供全双工通信，也就是发送端和接收端同时实现发送缓存二号接收缓存

  • TCP提供紧急flagURG 和push flag,PSH

  • TCP头部固定为（2^16-1-20）byte 的IP payload

  • TCP头部信息是20B

TCP报文头部

  • ACK代表 ack number是不是有效的 （处理第一次握手，以后全部的连接都是ACK=1）

  • SYN代表用来建立连接使用的

  • FIN 用来释放连接，FIN=1表示数据已经发送完毕，同时要去释放传输连接

  • 校验和是头部和数据两个部分，和UDP一样，IP是头部校验和

  • 可选项

    • MSS最大报文段长度
    • Windows scale size 窗口大小

TCP连接管理

1.三次握手

  • SYN在连接前两次出现
  • ACK在除了第一次以外的所有连接中出现
  • SYN-SEND->SYN-RSVD

2.四次挥手

  • FIN=1表示数据已经发送完毕，同时要去释放传输连接
  • ACK一直都是1
  • 注意FIN-WAIT-1 FIN-WAIT-2 2MSL

TCP流量控制

1.1流量控制和拥塞控制区别

拥塞控制：拥塞控制是作用于网络的，它是防止过多的数据注入到网络中，避免出现网络负载过大的情况；常用的方法就是：

  • （ 1 ）慢开始、拥塞避免
  • （ 2 ）快重传、快恢复。

流量控制：流量控制是作用于接收者的，它是控制发送者的发送速度从而使接收者来得及接收，防止分组丢失的。

1.2 TCP如何实现流量控制

  • 由滑动窗口协议实现。滑动窗口协议既保证了分组无差错、有序接收，也实现了流量控制。

  • 主要的方式就是接收方返回的 ACK 中会包含自己的接收窗口的大小，并且利用大小来控制发送方的数据发送。

  • 也就是数据链路层和窗口是不能变的，但是传输层缓存窗口大小是动态变化的

发送端的发送窗口大小 = min{ 拥塞窗口，接收方的接收窗口}

1.3 流量控制引发的死锁？怎么避免死锁的发生？

死锁问题：

  • 当发送者收到了一个窗口为0的应答，发送者便停止发送，等待接收者的下一个应答。但是如果这个窗口不为0的应答在传输过程丢失，发送者一直等待下去，而接收者以为发送者已经收到该应答，等待接收新数据，这样双方就相互等待，从而产生死锁.

解决方法：

  • 为了避免流量控制引发的死锁，TCP使用了持续计时器。每当发送者收到一个零窗口的应答后就启动该计时器。
  • 时间一到便主动发送报文询问接收者的窗口大小。
    • 若接收者仍然返回零窗口，则重置该计时器继续等待
    • 若窗口不为0，则表示应答报文丢失了，此时重置发送窗口后开始发送，这样就避免了死锁的产生。

相关定义

1.1性质

对于任意一个时刻，设f(u,v)实际流量，则整个图G的流网络满足3个性质：

  • 容量限制：对任意u,v∈V，f(u,v)≤c(u,v)。
  • 反对称性：对任意u,v∈V，f(u,v) = -f(v,u)。从u到v的流量一定是从v到u的流量的相反值。
  • 流守恒性：对任意u，若u不为S或T，一定有∑f(u,v)=0，(u,v)∈E。即u到相邻节点的流量之和为0，因为流入u的流量和u点流出的流量相等，u点本身不会”制造”和”消耗”流量。

1.2可行流

  • 弧流量限制条件: 0<=f(u,v)<=c(u,v);
  • 平衡条件:即流入一个点的流量要等于流出这个点的流量,(源点和汇点除外).

Prim算法

  • 选取一个顶点作为已选择的顶点，加入已选结点集合

  • 选出未选结点集合中选出一个和已选结点集合中的距离最小的结点

  • 该结点加入已选结点集合

证明：

用归纳法（反证法）证明形成的MST

  • 假设一个最短，但是不是最优的结点，和另外一个最优但是不是最短的边，
  • 两个不可能同时存在，必然只能选择一个
  • 明显最短的那个就是最优的。

效率：

  • 权重矩阵表示： $O(|V|^2)$
  • 邻接链表表示： $(|V|-1+|E|)O(log|V|)=O(|E|log|V|)$

  • 最后求出来的最小值大于等于inf，那么就是不存在最小生成树

Kruskal

思想：对边的操作

  • 将所有的边按照权重来进行排序

  • 按照排序的顺序依次判断排序好的边是否可以加入结果中（是否构成回路）

  • 难点：需要检查是否属于一个回路

用查并集来判断是否两个结点属于同一个树

实现

如何判断是否存在最小生成树

  • 采用Kruasal，最后所有的点是否都被包含进一个类别即可
  • 先判断连通性，由于结点数比较少，可以通过dfs直接判断连通块的个数

如何判断最小生成树是不是唯一的呢

  • 如果全部的边都是不一样的，那么生成树就是唯一的。
  • 无向图中存在相同权值的边是最小生成树不唯一的必要条件（但不是充分条件）

判定最小生成树唯一的算法思路

  • 求次小生成树，判断次小生成树的大小是否等于最小生成树

次小生成树

  • 求第二小生成树的时候，一般采用的是求出最小生成树后，
  • 依次删除最小生成树上的每一条边，然后生成n-1次最小生成树，
  • 记录下这个过程中的最小的生成树的值，那么这个就是第二小生成树了。
