1192. 查找集群内的「关键连接」

力扣数据中心有 n 台服务器，分别按从 0 到 n-1 的方式进行了编号。

它们之间以「服务器到服务器」点对点的形式相互连接组成了一个内部集群，其中连接 connections 是无向的。

从形式上讲，connections[i] = [a, b] 表示服务器 a 和 b 之间形成连接。任何服务器都可以直接或者间接地通过网络到达任何其他服务器。

「关键连接」是在该集群中的重要连接，也就是说，假如我们将它移除，便会导致某些服务器无法访问其他服务器。

请你以任意顺序返回该集群内的所有 「关键连接」。

Taijan算法求得的环路id实际上就是数据结构书中的Low[]数组记录的内容，实际上是一个东西。

关键注意点

  • 时间戳
    • 是用来标记图中每个节点在进行深度优先搜索时被访问的时间顺序，当然，你可以理解成一个序号（这个序号由小到大），用 dfn[x] 来表示
    • 实际上就是dfs树的id,就是num[]
  • 追溯值
    • 追溯值用来表示从当前节点 x 作为搜索树的根节点出发，能够访问到的所有节点中，时间戳最小的值 —— low[x]
    • 当前的low[v]的确定
      • 直接父节点+1
      • 相邻结点id计算好的
      • 相邻结点没有计算好的，用dfs先计算
    • 取三种情况的最小值

说白了，就是判断当前结点的邻接结点是不是id比他小，是就取小的值

  • 无向图的桥判定法则
    • 判断边 e （其对应的两个节点分别为 u 与 v）是否为桥，需要其满足如下条件即可：dfn[u] < low[v]

P问题，NP问题，NP完全问题

  • P :
    • 在多项式时间内解决的问题
    • 能在多项式时间内求解的判定问题
    • 并不是所有的判定问题都能在多项式时间内解决，例如：图灵停机问题
  • NP
    • 不能再多项式时间内解决，但是可以在多项式时间内验证该解是不是对的问题

p问题是NP问题的子集，以为NP问题还需要猜，然后验证，但是P问题可以确定解，并在多项式时间内验证

  • NPC
    • 可以把D1判定问题用多项式转化为D2判定问题
    • 每一个NP问题都能在多项式时间内化简为问题D,这个问题就是NPC问题
    • 背包问题，哈密顿都是NPC问题
    • 判断一个问题是NPC问题？
  • NP hard
    • 可以不是判断问题

  • 产生任意一个串，把他作为问题的待选解

三种编码定理总结

1.信源编码定理

​ 用典型列的语言：对于熵率H的稳定平稳信源，在所有长度为n的信源序列中，存在一个大小约为$2^{nH}$ 的子集，这个子集中序列出现的概率几乎相等，这些序列出现的概率和接近1.如果用每信元符号$R_{s}(bit)$ 的速率进行无损压缩编码，只$R_{s}>H$,则信源编码误差随$n->inf$而趋于0.

2.信道编码定理

信道编码

​ 码长长度很长时，只要满足传输速率$R_{c}<C$,传输误码率接近于0

3.信源信道分离定理

​ 只要满足$H<R_{s}<R_{c}<C$,就可以实现信源，信道分开编码实现误码率接近于0

​ $R_{s}=n00log(D)$

​ 如果要使得系统误码率为0，就必有$C>H_{inf}$ 因此联合编码无法改变香农极限，但是可以改进性能。
