希尔排序

  • 希尔排序的准确性是靠最小的间隔为1来保证的
  • 间隔是从大到小来实现的，也就是N/2,…1，原因在于，实现$h_{k-1}-sort$不会破坏$h_{k}-sort$的排序结果
  • 每用一个间隔以后，在实现一个插入排序
  • 希尔排序是非稳定排序算法
  • 采用希尔的序列，最差时间复杂度是$O(n^2)$ Hibbard的增量序列，$O(N^{2/3})$
  • 是一个inplace的算法，在线算法

（注：为方便记忆算法，我习惯将其记作“三层for循环+if” ——** for（for（for（if）））**）

1192. 查找集群内的「关键连接」

力扣数据中心有 n 台服务器，分别按从 0 到 n-1 的方式进行了编号。

它们之间以「服务器到服务器」点对点的形式相互连接组成了一个内部集群，其中连接 connections 是无向的。

从形式上讲，connections[i] = [a, b] 表示服务器 a 和 b 之间形成连接。任何服务器都可以直接或者间接地通过网络到达任何其他服务器。

「关键连接」是在该集群中的重要连接，也就是说，假如我们将它移除，便会导致某些服务器无法访问其他服务器。

请你以任意顺序返回该集群内的所有 「关键连接」。

获得割点

流程：

  • DFS获得生成图，并进行排序，生成的是一个生成树，用到了所有的结点，又没有环的存在，以为访问过的结点不会再被访问

任何的左右子树之间，都不会存在祖先孩子关系。分支和分支之间，不存在虚线边

因此剩下的边都是构成祖先孩子关系的

u是v的祖先，一定有：Num(u)<Num(v)

  • 根结点如果有多个分支子树，那么他一定是割点，因为他的分支之间，不存在虚线边

  • 判断根节点：只要有两个分支以上，根节点就是关键点

  • 如果一个点是关节点，

    • 至少有一个孩子
    • 同时不可能向下走，然后跳到他的祖先
    • 也就是必须他的后辈有一个连接到他的祖先，不然就是关节点

Low代表的就是，这个子树能连到的最高的祖先是哪个

以他为根的子树，能走到最高的点是哪个，需要后序的处理。

DFS找环(Taijan算法)

Taijan算法求得的环路id实际上就是数据结构书中的Low[]数组记录的内容，实际上是一个东西。

关键注意点

  • 时间戳
    • 是用来标记图中每个节点在进行深度优先搜索时被访问的时间顺序，当然，你可以理解成一个序号（这个序号由小到大），用 dfn[x] 来表示
    • 实际上就是dfs树的id,就是num[]
  • 追溯值
    • 追溯值用来表示从当前节点 x 作为搜索树的根节点出发，能够访问到的所有节点中，时间戳最小的值 —— low[x]
    • 当前的low[v]的确定
      • 直接父节点+1
      • 相邻结点id计算好的
      • 相邻结点没有计算好的，用dfs先计算
    • 取三种情况的最小值

说白了，就是判断当前结点的邻接结点是不是id比他小，是就取小的值

  • 无向图的桥判定法则
    • 判断边 e （其对应的两个节点分别为 u 与 v）是否为桥，需要其满足如下条件即可：dfn[u] < low[v]

网络层功能

  • 路由器

    • 路由器可以分割广播域，也就是不转发目的地址为255.255.255.255的目的IP包，可以分割广播域，避免广播风暴
    • 交换机工作在链路层，能分割冲突域，但是没办法分割广播域
    • 路由器连接的路由协议不一定是一样的，IPv4 IPv6

  • 存储转发机制

    • 路由器先接受完整的分组，然后检测分组是否有错，有错就丢弃，没有就等待路由算法提供路径

路由算法

1.分类

  • 静态路由算法（non-adaptive）
  • 动态路由算法 (adaptive)
    • 距离向量路由算法
    • 链路状态路由算法

2.距离向量算法（RIP）

  • 其中的距离常常用跳数来代表
  • 条数大于等于15表示不可达

3.链路状态算法（OSPF）

  • 最短路算法更新：dijkstra
  • 利用foold技术更新链路状态，同时具有快速收敛的特点
  • 对比:
    • 距离向量算法：仅仅和自己的邻居进行交谈
    • 链路状态：广播分组，获得所有的结点状态

4.层次路由

互联网将路由协议分为两部分：

  • 内部网关协议（IGP）,主要是在一个自治系统内部使用的协议，包括RIP,OSPF
  • 外部网关协议（EGP）,主要包括：BGP

5.路由表是什么

路由表建立的主要目标是为了实现路由协议和静态路由选择。

（1）链路层协议发现的路由（即是直连路由)

（2）静态路由

（3）动态路由协议发现的路由

  • 添加静态路由时，下一跳地址是填写本路由器的端口地址，还是对方路由器的端口地

    址？或者是目的地网络的路由器端口地址？下一跳地址写对方路由器的端口地址

  • 在同一个局域网内的2台PC机，IP地址分别为10.0.0.x/24和10.0.1.x/24，都属于VLAN1，一开始不能互相Ping通，为什么把子网掩码长度从24位变成16位，就通了？

    答：当掩码长度为 24 位时，两台 PC 不在同一网段，计算机首先查找网关的 MAC，如果网关 MAC 得不到回应，是 ping 不通的。当掩码长度为 16 位时，两台 PC 在同一网段，不需要通过网关，因此可以 ping 通。

思路

  • 用 dp[i][j] 表示以 (i, j) 为右下角，且只包含 1 的正方形的边长最大值

  • 递推关系很难得到，需要记忆，因此单独拿出来一个专题

  • 区别：

    • 和思考的区别在于利用dp[i][j]来固定了右下角的元素，矩阵的右下角元素一定是[i,j]

P问题，NP问题，NP完全问题

  • P :
    • 在多项式时间内解决的问题
    • 能在多项式时间内求解的判定问题
    • 并不是所有的判定问题都能在多项式时间内解决，例如：图灵停机问题
  • NP
    • 不能再多项式时间内解决，但是可以在多项式时间内验证该解是不是对的问题

p问题是NP问题的子集，以为NP问题还需要猜，然后验证，但是P问题可以确定解，并在多项式时间内验证

  • NPC
    • 可以把D1判定问题用多项式转化为D2判定问题
    • 每一个NP问题都能在多项式时间内化简为问题D,这个问题就是NPC问题
    • 背包问题，哈密顿都是NPC问题
    • 判断一个问题是NPC问题？
  • NP hard
    • 可以不是判断问题

不确定多项式类型：验证问题是多项式问题

  • 产生任意一个串，把他作为问题的待选解
  • 在多项式时间内对问题进行判断
