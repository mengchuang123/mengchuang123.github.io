1631. 最小体力消耗路径

你准备参加一场远足活动。给你一个二维 rows x columns 的地图 heights ，其中 heights[row]``[col] 表示格子 (row, col) 的高度。一开始你在最左上角的格子 (0, 0) ，且你希望去最右下角的格子 (rows-1, columns-1) （注意下标从 0 开始编号）。你每次可以往 上，下，左，右 四个方向之一移动，你想要找到耗费 体力 最小的一条路径。一条路径耗费的 体力值 是路径上相邻格子之间 高度差绝对值 的 最大值 决定的。

请你返回从左上角走到右下角的最小 体力消耗值 。

示例一：

想法：

  • 到点dp[i][j] 的所有路径集合中的最小值划分依据：最后一步是哪里来的dp[i][j] = min( max(dp[i-1][j],|g[i][j]-g[i-1][j]|),... )—-似乎无法用DP来求解.
  • 当题目允许往任意方向移动时，考察的往往就不是 DP 了，而是图论从本质上说，DP 问题是一类特殊的图论问题。
  • 对于不是拓扑图的图论问题，我们无法使用 DP 求解。而此类看似 DP，实则图论的问题，通常是最小生成树或者最短路问题。

思路：

  • 如何将二维矩阵转化为顺序序列

  • 记住对于row * col的矩阵，任意的点 (i,j) 可以使用 i * col + j 转化为顺序编码idx（等价于从上往下 从左到右编号）

    反过来将 idx 恢复成 (i,j) 表示只需要 (idx / col, idx % col ) 即可

  • 超过内存限制。。。。。无奈
